This function generates double precision floating point pseudorandom numbers which distribute in the range \mbox{[}1, 2) to the specified array\mbox{[}\mbox{]} by one call. The number of pseudorandom numbers is specified by the argument {\bfseries size}, which must be at least (S\+F\+M\+T\+\_\+\+M\+E\+X\+P / 128) $\ast$ 2 and a multiple of two. The function get\+\_\+min\+\_\+array\+\_\+size() returns this minimum size. The generation by this function is much faster than the following fill\+\_\+array\+\_\+xxx functions.

For initialization, init\+\_\+gen\+\_\+rand() or init\+\_\+by\+\_\+array() must be called before the first call of this function. This function can not be used after calling genrand\+\_\+xxx functions, without initialization.


\begin{DoxyParams}{Parameters}
{\em dsfmt} & dsfmt state vector. \\
\hline
{\em array} & an array where pseudorandom numbers are filled by this function. The pointer to the array must be \char`\"{}aligned\char`\"{} (namely, must be a multiple of 16) in the S\+I\+M\+D version, since it refers to the address of a 128-\/bit integer. In the standard C version, the pointer is arbitrary.\\
\hline
{\em size} & the number of 64-\/bit pseudorandom integers to be generated. size must be a multiple of 2, and greater than or equal to (S\+F\+M\+T\+\_\+\+M\+E\+X\+P / 128) $\ast$ 2.\\
\hline
\end{DoxyParams}
\begin{DoxyNote}{Note}
{\bfseries memalign} or {\bfseries posix\+\_\+memalign} is available to get aligned memory. Mac O\+S\+X doesn\textquotesingle{}t have these functions, but {\bfseries malloc} of O\+S\+X returns the pointer to the aligned memory block. 
\end{DoxyNote}
